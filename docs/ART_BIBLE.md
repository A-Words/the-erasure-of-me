# 《记忆的缝隙》美术与资产规范

> 版本：v0.1
> 风格：低饱和手绘 2D、温暖现实与轻度超现实
> 网格：64×64 逻辑像素
> 设计分辨率：1280×720

## 1. 视觉目标

### 1.1 关键词

旧纸、晨光、雨水、生活磨损、克制留白、褪色红伞。

### 1.2 应当呈现

- 普通家庭真实生活过的痕迹；
- 柔和但不甜腻的暖色；
- 通过褪色、缺口、重叠和透视轻微错位表现记忆变化；
- 红伞作为稳定、易识别的情感锚点；
- 角色保有尊严和行动主体性。

### 1.3 禁止方向

- 恐怖医院、阴森怪物、黑色触手、血污或惊吓构图；
- 把老人画成儿童比例或滑稽佝偻；
- 使用过量故障艺术、强色散和快速闪烁；
- 将红色广泛用于错误、危险或按钮警告；
- 过度干净的样板间和通用 SaaS 风格 HUD。

## 3. 字体与文字

- UI 与字幕：Noto Sans SC；
- 日记与记忆标题：Noto Serif SC；
- 字体以 OFL 许可版本自托管；
- 正文默认 20 CSS px，最低 18 px；
- 大字体模式 24 px；
- 字幕行高 1.5，正文行高 1.6；
- 不用手写字体承载重要信息；
- D2 的缺字由遮罩或替代字符生成，原文始终保留给辅助技术。

## 4. 环境资产

### 4.1 Tilemap

- 逻辑格 64×64；
- 绘制源文件建议 128×128 后按 2 倍缩小导出；
- 地面 tile 可无缝拼接；
- 墙体和高家具允许跨 1×2 或 2×2 格；
- 碰撞边界按逻辑格和 Tiled 多边形定义，不从透明像素自动推断；
- 前景遮挡物必须提供角色进入后的透明淡化状态。

### 4.2 房间构图

- 主路径宽度至少 2 格；
- 门口净宽至少 1.25 格；
- 可交互物与纯装饰物在轮廓或微动上可区分；
- 每个房间最多一个强暖色焦点；
- 中央和角色脚下不放持续高对比 UI 或装饰噪点。

### 4.2.1 交互点表现

- 不显示常驻大圆按钮或图标；可交互对象以场景内物品/区域本身为视觉提示。
- 鼠标悬停（或聚焦）时才显示对象名称标签。
- 静止状态只保留一个几乎不可见的小点（半径约 6 逻辑像素）作为热区锚点，悬停时稍微放大并变为半透明。
- 标签使用暖白文字（#F7F3E8）配墨色半透明背景（#2F2B28CC），15 px Noto Sans SC，位于对象下方，避免遮挡角色。
- 减少动态效果模式下，标签与小点直接切换透明度，不执行缩放或循环动画。
- 纯键盘路径仍通过近距离 HUD 提示（下中情境交互提示）提供可交互信息，不依赖悬停。
- 有精灵的可交互物（床头合影、红线日记、钥匙碗、眼镜盒、红伞等）在静止时带轻微缩放呼吸微动（约 ±3.5%、2.4 秒正弦、相位错开），以轮廓微动区别于纯装饰物；无精灵的交互点（出口/门）由热区小点做轻微 alpha 与缩放呼吸（alpha 0.12–0.25、缩放 0.88–1.12）。呼吸仅作可发现性提示，不承担目标引导（仍由 90 秒/150 秒分级提示负责）。秀兰不参与呼吸。减少动态效果模式下两者均关闭，回归"近距 HUD 提示 + 悬停标签"通道。

按住 Shift 的“静静留意”不是扫描或侦测技能。标准模式以角色为中心显示一次扩散到 192 逻辑像素的克制圆纹，同时用章节色薄幕压低远处画面噪声，只加强近距内当前已经可见的实体轮廓和既有章节线索；不得用常驻光柱、答案连线、文字列表或新生成图标揭示解法。松开按键、移动、打开对白或 DOM 面板即撤除。减少动态效果模式固定显示 192 像素静态边界、静态薄幕、实体轮廓和角色注意姿态，不播放扩散、呼吸或位移动画。

### 4.2.2 有效进展的场景回应

每次有效进展使用发生位置附近的细环、短线碎片或材质轻亮完成 0.6–1.2 秒回应，并与短三音锚点同步。普通步骤保持较小范围，路口完成、记忆片段和牵手等大节点可以稍宽但不得盖住 HUD。回应禁止使用奖章、星级、分数、连击、全屏礼花或“记忆恢复”字样；颜色沿用当前章节色，红色只可在红伞本身或既有情感锚点上出现。减少动态效果模式让静态轮廓保持 600 毫秒，不做扩散或碎片位移。

### 4.3 记忆变体

同一空间的年代变化优先复用结构，通过以下内容区分：

- 家具新旧、纸箱、窗台植物高度；
- 墙面颜色与生活痕迹；
- 光照方向和环境粒子；
- 道具替换与局部遮罩。

不得为了“记忆错乱”改变可行走区域而不更新导航和视觉边界。

环境候选以原创 SVG 或归档的高分辨率生成原稿保存于 `assets-source/art/environments/`，通过 `npm run assets:render:environments` 统一导出 1280×720 运行时 PNG。背景只承担地板、墙体、门窗洞口和中性环境光；出生点、交互对象和稳定 ID 仍由 Tiled JSON 对象层提供。可变化的直射光、窗格影子和光斑必须使用独立 overlay，不烘焙进背景。

第三章 Shared Life v02 以“同一个家在三个年代同时显影”为构图基线。1979、1992、2001 三个区域各有一扇同源四格木窗：外框尺寸、窗格比例、中央插销与右下磕痕保持一致，只改变木框磨损、玻璃光线和生活痕迹。左侧保留纸箱与镜台，中段以桂花窗台、身高刻度和圆形水印形成最暖区域，右侧以收音机与停电纪念日晚餐形成柔和夜景；下中出生区、绕行相册桌的主通道与上中延长走廊必须始终清楚。场景中不悬挂年份牌；准确年份只在可检查的照片线索中以 DOM 可控排版显示，不写入生成背景或照片像素。

Shared Life 背景不得烘焙三张松散照片、木梳、搪瓷杯、录音带或空相册。上述对象继续使用透明图集；归位前显示条纹托盘、圆点垫和波纹凹槽，归位后同一实体视图切换为对应实物 frame。三段年代叠影只做低透明度淡变：照片排序完成后先降低全部接缝薄雾，每归位一件物品再收束对应年代；三件全部归位后，以 `environment.life.resolved` 短淡变叠入连贯的当代客厅印象。收束态与初始态保持墙体、走廊、中央桌和固定家具注册一致，只保留中间的成熟桂花窗，左右窗、搬家纸箱、年代接缝和停电宴席自然淡回同一晨光下的连续墙地与日常圆桌。减少动态效果下所有阶段统一使用 140 毫秒短淡变。

照片排序使用 DOM 相册页，不使用通用表格或纯文字清单：三张 v02 照片放入旧纸卡与浅色相纸边框，顶部编号和横向缝线表达“从最早到最近”，年份、生活片段名称和非文字线索都保持真实文本。每张卡只提供“早一些 / 晚一些”两个同等可键盘访问的按钮，禁用已经越界的方向；窄窗口允许面板滚动，但不裁掉照片、线索或确认按钮。

雨站 v02 背景使用与 home v10 接近的 30°～35° 轻俯视空间语言，并参考 `memory_rain_umbrella_v01.png` 的旧车站、钟表铺橱窗、红伞记忆和湿石材情绪：站台边缘、售票亭、雨棚、钟表铺立面和屋檐必须有可见厚度与接触阴影，湿石材路径保持从左下到右上的清晰通行关系。底图避免过暗或黑块，雨线与地面积水反光拆为 `environment.rain.rain_overlay` 和 `environment.rain.puddle_reflection_overlay` 透明层；标准模式下雨线层以持续下落循环形成动态降雨，减少动态效果模式下保留静止、低透明度雨幕。车票、2/4/5 石板、红伞招牌和钟表铺前红伞继续作为独立 visual_props，不烘焙进背景或天气层。

第四章 v02“缝隙长廊”使用 1280×720 的空十字构图：中央和上下通道借用清晨住宅的旧木地板与墙体，左侧使用雨站的湿深色石板，右侧使用共同生活章节的浅色磨损花砖，使玩家在一个空间内感到三个熟悉地点互相缝合。交叉口中央必须保持低噪、四出口轮廓等权，底图不得烘焙箭头、红伞方向、哼唱波纹或脚印。运行时只显示当前状态派生的下一步线索：地面箭头、伞影、哼唱波纹、分级脚印或完成后的家门；红伞色永远对应正确世界方向，错误出口不使用伞形或锚点红装饰。

尾声 v02 使用 1280×720 温暖清晨住宅：中央客厅和牵手构图区开放、没有高对比杂物；圆桌上固定保留两碗热面，褪色红伞靠在右侧墙边，两侧窗光将视线缓慢带回中央。亮度恢复但饱和度不恢复，不加入医疗符号、奇迹光效、胜利构图或能被误解为“记忆恢复”的视觉隐喻。环境换景不得改动既有对白、牵手主动权、科普入口或核心四项医学结尾事实。

清晨住宅中的床边合影、红线日记、蓝色钥匙碗和眼镜盒使用独立透明 PNG，不烘焙进环境背景；运行时由 Tiled 稳定 ID 定位，拾取后随实体视图一起隐藏。

清晨住宅 v11 在 v10 正交结构上统一手绘材质、木地板尺度、墙面颗粒和接触阴影：主要外墙、隔墙、中墙和门洞边界继续保持水平或垂直，卧室、客厅、厨房、封闭储物间、走廊与右下玄关的结构和碰撞注册不变。木地板允许出现低对比磨损和行走痕迹，但不得形成伪路线或交互标记；厨房瓷砖与木地板通过同一描边、颗粒和环境色融合。卧室—客厅、客厅—厨房的上方开口、左下储物间通道与中央宽通道沿用 v10 尺寸。右上窗户继续使用四格木窗，不改变门窗数量或位置。家具保持 30°～35° 轻斜俯视和一致落地面，茶几与柜体之间保留连续可行走通道。

`environment.home.sunlight_overlay` 独立保存窗外阳光束、厨房地面的透视亮斑、窗棂影子、少量反射光斑，以及连接卧室、客厅和走廊的低透明度晨光反射，运行时置于背景之上、家具之下。overlay 使用静态透明 PNG，不包含交互提示；各房间的反射光不得抹平墙角接触阴影或抢过关键道具。中墙和前墙（含左下竖墙段）另由 v11 背景机械提取到现有 `environment.home.crosswall_overlay` 与 `environment.home.frontwall_overlay`，在人物/家具排序层中重绘，保证墙体视觉和遮挡像素完全一致；右侧门框和墙面只保留在背景中，不制作独立遮挡 overlay。上方两道隔墙为竖向墙体，角色不会走到其后方，也不使用遮挡 overlay。

床、边桌、沙发、木桌、厨房台柜、储物柜和玄关桌均为独立透明家具素材，由 Phaser 按场景坐标组合。沙发必须使用垂直约 35° 的轻斜俯视：两块靠背正面、短纵深坐垫、前沿木底座和前脚均需可见，不能退化成近乎完全俯视的四块软垫。入户门不是家具：v11 背景在右下玄关的右侧墙体中完整绘制墙、门洞、门框和门槛；当前出口交互不播放开门动画，也不要求角色走入门洞，因此不绘制独立门板 sprite，不制作右墙遮挡 overlay，只由室内侧门口 hotspot 触发。如果以后加入开关门演出，门板必须作为贴合该门框透视的独立 sprite，不能画成正对镜头的完整矩形或贴在画面边缘的竖条。渲染按脚底/家具底座的 Y 值排序：人物在底座排序线后方时被家具遮挡，越过排序线后覆盖家具；水平墙前脸可作为独立遮挡段参与同一排序。

非交互生活细节使用独立装饰层：床边旧条纹地毯、布拖鞋、厨房搪瓷暖水瓶/调味罐/旧砧板、茶几下低饱和旧地毯、玄关门垫和客厅旧木钟。茶几地毯负责把下方公共空间收拢成生活区域，不能遮住眼镜盒轮廓；玄关门垫只强化门口材质变化，不承担路线提示；旧木钟使用刻度而非数字或文字，保持普通家庭物件感。厨房杂物固定放在水槽与煤气炉之间的空余台面，不覆盖炉头。装饰不得发光、闪烁或伪装为谜题线索，不得包含照片、日记、眼镜盒、钥匙盘等关键交互道具。

### 4.4 清晨住宅比例标尺

住宅比例以老年角色、双人床和标准住宅门为三个锚点。运行时尺寸指透明素材的正方形显示画布；验收时还要检查画布内有效像素，不能只比较源文件尺寸。小型交互物允许在严格实物比例上放大约 15%～30%，但不得大到改变承载家具的用途或视觉重量。

| 对象 | 现实参考范围 | 运行时显示画布 | 比例要求 |
| --- | --- | ---: | --- |
| 老年角色 | 身高约 160～175 cm | 64×96 帧，住宅中 1.2 倍 | 全场景主比例锚点 |
| 双人床 | 约 150×200 cm | 225×225 | 有效床宽约为角色有效宽度的 4 倍 |
| 标准住宅门 | 高约 200 cm | 背景嵌墙门洞；当前无门板 sprite | 门框、门槛与右侧墙透视一致；hotspot 位于门洞内侧 |
| 床边桌 / 客厅边桌 | 宽约 45～55 cm | 64×64 / 58×58 | 桌面能承载小物且仍清楚可见 |
| 沙发 / 茶几 | 宽约 150～190 cm / 90～120 cm | 205×205 / 165×165 | 茶几有效宽度小于沙发有效宽度 |
| 厨房台柜 / 储物柜 / 玄关桌 | 宽约 90～190 cm | 220×220 / 160×160 / 100×100 | 高度和落地脚印与人物一致 |
| 床边地毯 / 茶几地毯 / 门垫 | 生活用品常规尺寸 | 160×160 / 230×230 / 100×100 | 茶几地毯宽于桌面但不封闭绕行通道；门垫保持玄关地面可辨 |
| 一双拖鞋 / 厨房杂物 / 旧木钟 | 生活用品常规尺寸 | 34×34 / 64×64 / 44×44 | 拖鞋整组有效宽度不超过角色身体；旧木钟在实际比例下只读轮廓与指针 |
| 相框 / 日记 / 钥匙碗 / 眼镜盒 | 桌面与手持物尺寸 | 36×36 / 44×44 / 32×32 / 38×38 | 相框有效宽度不超过床边桌约一半；其余仅轻微强调 |

上述运行时尺寸统一由 `src/game/content/homeLayout.ts` 的 `homeVisualSizes` 提供；Scene、碰撞作者数据和资产登记不得各自维护另一套比例。

## 5. 角色规格

### 5.1 尺寸与锚点

| 资产 | 游戏帧 | 绘制源 | 锚点 |
| --- | --- | --- | --- |
| 老年许志远 | 64×96 | 128×192 | 底部中心 |
| 老年林秀兰 | 64×96 | 128×192 | 底部中心 |
| 年轻角色 | 64×96 | 128×192 | 底部中心 |
| 手部尾声特写 | 512×256 | 1024×512 | 画面中心 |

碰撞体采用脚下 32×20 椭圆或矩形，不按全身边界碰撞。

住宅章节中的老年许志远视图使用 1.2 倍显示缩放，脚部可见像素与 32×20 碰撞体底边对齐，领域坐标和碰撞体保持不变；该缩放用于修正人物与床、厨房台柜及茶几的视觉比例，不改变移动速度和交互距离。

### 5.2 动画

| 动画 | 方向 | 帧数 | 建议帧率 |
| --- | --- | --- | --- |
| idle | 4 | 4 | 4 FPS |
| walk | 4 | 6 | 8 FPS |
| observe | 4 | 4 | 6 FPS |
| pickup | 下/侧 | 6 | 8 FPS |
| reach_hand | 侧 | 8 | 8 FPS |
| hold_hand | 侧 | 4 | 4 FPS |

许志远步态稍慢但稳定，不加入喜剧性踉跄。林秀兰的伸手动作必须包含“停下等待”的前置帧。

拾取使用向下与右侧两条 6 帧资产：上下朝向共用向下条带，左右朝向共用右侧条带并镜像。减少动态效果下不播放完整屈膝过程，只短暂显示一个静态俯身姿态。

牵手使用 4 帧 512×256 侧向特写，按“等待—靠近—初触—轻握”顺序由领域长按进度选择帧；取消后回到等待画面。减少动态效果下只显示静态初触帧，完成后切入尾声正式特写。

### 5.3 精灵制作流程

1. 先制作一张在游戏实际比例下通过评审的种子帧；
2. 以种子帧为身份、服装、比例和色彩基准；
3. 同一动作一次制作完整横向动画条，不逐帧独立生成；
4. 整条动画使用同一缩放比例；
5. 所有帧统一底部中心锚点；
6. 必要时把第一帧锁回已经批准的种子帧；
7. 输出透明 PNG 和预览表；
8. 在 Phaser 灰盒中检查后才能登记到 manifest。

使用生成式工具时，提示必须明确：同一角色、同一朝向、同一轮廓与服装、准确帧数、透明背景、无文字、无场景、非海报构图。生成结果必须经过人工修整和授权记录。

### 5.4 精灵验收

- 各帧头顶高度与脚底位置无漂移；
- 角色朝向和动作在 100% 游戏比例可读；
- 四方向服装细节不左右跳变；
- 透明边缘无白边或脏像素；
- idle 到 walk 的第一帧无尺寸跳变；
- 预览动画通过后再修改资产索引。

## 7. UI 视觉

### 7.1 材质

- 面板像半透明旧纸或磨砂玻璃上的纸片；
- 边框使用不完全闭合的细线；
- 焦点框使用焦点青和 2–3 px 实线；
- 阴影柔和，不使用厚重卡片堆叠；
- 图标保持单色轮廓并配文字标签。

### 7.2 HUD 预算

- 左上目标：最多一行，四秒后收起；
- HUD 操作栏：背包 / 日记 / 地图 三个文字按钮并排，地图入口在 D4 随 HUD 淡出且不渲染；
- 展开地图使用同一纸面 SVG 语言：细线表示道路和房间边界，焦点青双圆点表示玩家，红伞保持暗红图形，已到站牌用带数字的实心圆表示；
- D1 水渍只能降低未停留道路和区域文字的清晰度，不遮盖玩家、已到站牌、红伞和钟声波纹；
- 左下背包、右下日记：只保留一个小型入口；
- 下中交互提示：只在靠近物件时出现；
- 常驻覆盖不超过 20%，中央与下中游戏区域保持清晰。

### 7.3 退化表现

| 阶段 | 允许效果 | 禁止效果 |
| --- | --- | --- |
| D1 | 水渍遮罩、局部文字淡出 | 整张地图持续晃动 |
| D2 | 缺字、轻微基线偏移 | 高频乱码闪烁 |
| D3 | 指南针和按键图标一次旋转 | 不断随机重排 UI |
| D4 | HUD 纸屑般淡出 | 隐藏暂停和字幕 |

减少动态效果模式将所有扩散、漂移和位移动画替换为 100–200 毫秒淡变。

### 7.4 菜单与弹窗

- “读取记忆”中同一片段的“读取”和“删除”按钮必须等宽、等高；窄视口可扩展到整行，但不得因按钮语义不同产生尺寸跳变；
- 二次确认弹窗的主操作与返回按钮使用等宽布局，按钮间距不小于 0.75 rem；空间不足时整组换行，不允许按钮相贴；
- 标题页设置按“声音”和“显示与操作”分组，桌面端双列、窄视口单列；控件继续使用深色旧纸/磨砂材质，不混入突兀的浅灰面板；
- 桌面高度足够时设置面板不显示内部滚动条；低高度或窄视口由标题页外层自然滚动，返回入口保持在设置内容之后。
