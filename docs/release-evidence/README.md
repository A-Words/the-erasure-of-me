# v0.1 外部测试与发布证据包

本目录把 GDD 第 16、17 节的外部门槛转换为统一记录。模板本身不是完成证据；只有真实执行、匿名记录并通过汇总脚本后，才能更新 `RELEASE_READINESS.md`。

## 1. 使用方式

1. 复制 `templates` 到仓库外的私密工作目录，例如 `E:\playtest-data\memory-seams-v0.1`。
2. 不改列名，按 UTF-8 CSV 填写五张表；一行只记录一次真实会话、检查、审核或决定。
3. 不收集姓名、联系方式、健康史、诊断、自由按键序列或其他非必要个人信息。
4. 执行 `npm run release:evidence -- <证据目录> --output <汇总报告路径>`。
5. 只有脚本结论为 `PASS`，且人工确认原始记录与签署材料真实有效后，才可关闭相应发布门槛。

建议把生成的匿名聚合报告提交到仓库；原始记录和审核者联系方式仅在取得同意且确有必要时保存。

## 2. 文件说明

| 文件 | 一行代表 | 核心门槛 |
| --- | --- | --- |
| `playtest-sessions.csv` | 一名首次玩家的一次完整试玩 | 至少 15 人；80% 在 20–30 分钟内无攻略完成；三级提示、D3、设置与主题理解达标 |
| `accessibility-checks.csv` | 一条独立无障碍路径的完整通关或菜单检查 | 纯键盘、静音、颜色冗余、低扰动、减少动态、200% 缩放、屏幕阅读器全部通过 |
| `browser-checks.csv` | 一个浏览器/系统/视口组合的发布构建检查 | Chrome、Edge、Firefox 与 macOS Safari 无阻塞问题 |
| `external-reviews.csv` | 一名审核者的一轮正式结论 | 至少一名医学专业人员和一名家属/照护者；阻塞与高优先级问题清零 |
| `release-decisions.csv` | 项目负责人对一个发布范围事项的书面决定 | 真人语音范围、项目许可证和资产权利状态都有明确结论 |

## 3. 试玩主持规则

- 使用未接触过本作的参与者；至少包含部分 45 岁以上家庭成员。
- 先展示内容提示与退出方式，再取得测试同意；参与者可随时停止且无需解释。
- 主持人只处理设备故障和安全问题，不解释谜题。玩家主动要求提示或主持人提供攻略时，将 `external_help` 记为 `yes`。
- 从标题页开始计时，到科普页出现结束；休息、设备故障和访谈时间不计入 `duration_minutes`。
- `*_hint_level` 记录该谜题实际出现的最高提示等级；未触发填 `0`。
- `d3_first_junction_attempts` 记录教学后首次路口的尝试次数。
- `build_commit` 填本次候选构建的完整或短 Git 提交号；同一轮 15 人测试只使用一个冻结构建。
- 五个主题问题必须在主持人解释主题前完成；`theme_understood` 只有在玩家明确理解“疾病未被治愈，但陪伴仍有价值”时填 `yes`。
- 若玩家认为小游戏能诊断认知健康，立即把 `q2_diagnostic` 记为 `yes` 或 `unclear`，暂停该版本的对外测试并修订文案。
- 患者或照护者参与时使用单独、知情且有支持的流程，不把他们作为压力测试对象。

## 4. 无障碍与浏览器检查

- 每个 `accessibility_track` 必须从标题到科普页独立完成；`zoom_200` 和 `screen_reader` 可只验证 GDD 指定的菜单、内容提示、目标文字、设置和科普页，但需在 `notes` 写明范围。
- `color_redundancy` 使用系统或浏览器的常见色觉差异模拟，确认形状、纹理和文字仍能区分线索。
- `browser-checks.csv` 的 Safari 必须来自真实 macOS Safari；Windows WebKit 自动化不能替代。
- 发现问题时先创建可追踪的问题 ID，再把 ID 写入 `blocker_id`。阻塞问题未关闭时状态不得填 `pass`。

## 5. 审核与隐私

- 医学审核者需在 `qualification` 记录专业领域，不要求在公开报告中披露姓名。
- 敏感性审核者只需记录与患者/照护经历的关系类别；不记录患者身份或诊断细节。
- 原始签署材料应保存日期、审核范围、结论和问题清单；是否公开姓名与致谢必须另行取得同意。
- `release-decisions.csv` 必须由有权决定发布范围和许可证的项目负责人填写。

## 6. 判定公式

- 无攻略准时完成率：`completed=yes`、`external_help=no` 且 `duration_minutes` 在 20–30 之间的首次玩家数 ÷ 合格首次玩家数。
- 三级提示率：每个主谜题中最高提示等级为 3 的玩家数 ÷ 合格首次玩家数。
- D3 两次内通过率：`d3_first_junction_attempts <= 2` 的玩家数 ÷ 合格首次玩家数。
- 主题理解率：`theme_understood=yes` 的玩家数 ÷ 合格首次玩家数。
- 任一玩家将小游戏理解为诊断工具、任一玩家被设置层阻塞、或存在未关闭的严重不良事件，均判定失败并要求修订后重新测试。

脚本负责计算，不允许手工四舍五入后宣称达标。
