# 《记忆的缝隙》关卡设计文档

> 版本：v0.1
> 地图工具：Tiled
> 网格：64×64 逻辑像素
> 相机设计分辨率：1280×720
> 对应 GDD：v0.1

## 1. 关卡设计原则

1. 每个章节只引入一种主要退化，不同时让玩家失去多个新能力。
2. 主要路径在不打开地图的情况下也能通过构图、物件、声音或纹理辨认。
3. 每个关键谜题至少包含两种不同类别的线索。
4. 走错或摆错不清空已经确认的正确步骤。
5. 检查点恢复后，角色必须位于安全导航区且面向当前目标。
6. 红伞只标记正确方向、关系记忆或安全空间，不用于误导。
7. 所有计时都用于提示升级，不用于倒计时失败。

## 2. 通用 Tiled 规范

地图图层、对象属性和命名以 TECHNICAL_DESIGN.md 为准。

### 2.1 坐标

本文坐标均为 Tiled 网格坐标，格式为 (x, y)，左上角为 (0, 0)。实体实际锚点为格子底部中心。

### 2.2 稳定 ID

| 前缀 | 用途 | 示例 |
| --- | --- | --- |
| map. | 地图 | map.home |
| spawn. | 出生点 | spawn.home.bed |
| entity. | 可见实体 | entity.home.key_bowl |
| interact. | 交互配置 | interact.home.key_bowl |
| trigger. | 一次性或区域触发 | trigger.home.thunder |
| puzzle. | 谜题 | puzzle.rain.station_sequence |
| checkpoint. | 存档点 | checkpoint.home.journal |
| dialogue. | 对白序列 | dialogue.home.wake |
| journal. | 日记页 | journal.home.key |
| memory. | 记忆碎片 | memory.rain.umbrella |
| audio. | 声音 | audio.rain.clock_bell |

### 2.3 通用交互距离

- 正面交互：角色锚点距离物件锚点不超过 96 像素；
- 观察高亮：不超过 192 像素且中间无遮挡；
- 门与出口：角色碰撞体进入 48 像素深的触发区域；
- 放置槽：打开物品选择面板后冻结移动。

## 4. 第一章：清晨的家

### 4.1 地图参数

- 地图 ID：map.home
- 尺寸：32×18 格
- 退化阶段：D0
- 出生点：spawn.home.bed = (5, 5)，朝下
- 相机：按房间 camera_zones 柔性切换

### 4.2 平面布局

~~~text
┌──────────┬────────────┬────────┐
│  卧室    │    客厅    │  厨房  │
│ P  照片  │ 日记 眼镜盒│        │
│          │            │        │
├──────┬───┴────────────┴────┬───┤
│储物间│       走廊           │玄关│
│封闭  │                      │碗 门
└──────┴──────────────────────┴───┘
~~~

储物间仅作为背景，不可进入，防止首章探索范围膨胀。

住宅建筑壳层采用水平方向正对、仅垂直方向约 35° 俯角的斜俯视；家具可按摆放方向采用合理转角，但必须共享一致的落地面和俯角观感。背景层只包含地板、墙体、房间开口、窗洞与晨光；床、床边桌、沙发、客厅边桌、椭圆木桌、厨房台柜、储物柜、玄关桌和入户门均由独立透明素材按坐标组合，不得烘焙回背景；入户门保持在画面右侧边。

### 4.4 流程节点

| 顺序 | 条件 | 行为 | 状态变化 |
| --- | --- | --- | --- |
| 1 | 章节开始 | 播放起床对白与移动教学 | flag.home.awake |
| 2 | 首次检查床边照片 | 教学观察和交互 | flag.tutorial.interact |
| 3 | 首次离开卧室 | 提示 M 打开地图 | flag.tutorial.map |
| 4 | 拾取眼镜盒或日记 | 教学 I 打开背包 | flag.tutorial.inventory |
| 5 | 检查日记 | 解锁 journal.home.key | flag.home.read_key_note |
| 6 | 拾取钥匙 | 写入 checkpoint.home.key | inventory.home_key |
| 7 | 持有钥匙和日记时开门 | 雷声、切换第二章 | chapter.rain |

玩家可以先取得钥匙再读日记；开门仍要求拾取日记，避免丢失后续核心功能。

### 4.5 检查点

- checkpoint.home.start：章节入口；
- checkpoint.home.journal：读完钥匙日记；
- checkpoint.home.key：取得钥匙。

### 4.6 提示升级

| 无进展时间 | 表现 |
| --- | --- |
| 90 秒 | 蓝色钥匙碗出现两次柔和反光 |
| 150 秒 | 日记入口发光，若已读日记则目标文字变为“看看玄关的小碗” |
| 210 秒 | 观察模式中显示从客厅到玄关的淡脚印 |

无进展定义：未首次检查新物件、未拾取道具、未进入新房间。

### 4.7 首章验收

- 新玩家能在 60 秒内理解移动与交互；
- 不打开地图也能通过房间构图找到玄关；
- 提前执行操作会跳过对应教学；
- 持有条件不完整时，门给出具体、非责备提示；
- 刷新后从最近检查点恢复。

| 红伞招牌一 | (27, 8) | entity.rain.umbrella_sign_a |
| 红伞招牌二 | (35, 6) | entity.rain.umbrella_sign_b |

### 5.4 D1 触发

玩家第一次打开地图时显示完整道路。关闭地图并向右移动超过两格后进入 trigger.rain.map_wash：

1. 冻结移动 1.2 秒；
2. 播放雷声与地图纸张受潮音；
3. HUD 地图边缘出现水渍；
4. 切换 D1；
5. 显示短句“有些路名看不清了。钟声还在。”

减少动态效果模式只做 200 毫秒淡变，不使用扩散动画。

### 5.5 谜题一：站牌序列

- 谜题 ID：puzzle.rain.station_sequence
- 答案：2 → 4 → 5
- 输入：角色走入石板区域即提交
- 重置：不清空正确前缀；踩错石板变暗 600 毫秒

状态：

~~~text
[] → [2] → [2,4] → [2,4,5] → completed
~~~

若当前前缀为 [2] 而玩家踩 5，状态保持 [2]。完成后钟表铺方向的雨幕减弱，并写入 checkpoint.rain.sequence。

### 5.6 谜题二：声音路线

- 谜题 ID：puzzle.rain.sound_route
- 起点：(25, 10)
- 正确地标：红伞招牌 A → 红伞招牌 B → 钟表铺红伞
- 声音：钟声每 6 秒三响，声像指向东北
- 视觉替代：屏幕东北边缘同心圆、积水波纹朝东北扩散

玩家触碰招牌 A 后，招牌 B 才出现暖色反光；触碰 B 后开放钟表铺门前导航区。

### 5.7 提示升级

| 谜题 | 等级 1 | 等级 2 | 等级 3 |
| --- | --- | --- | --- |
| 站牌 | 车票边缘发亮 | 下一个数字与圆点发亮 | 正确石板出现水面倒影 |
| 声音路线 | 钟声视觉波纹增强 | 当前红伞招牌轻摆 | 地面积水连续显示方向 |

### 5.8 记忆与出口

与 entity.rain.red_umbrella 交互：

1. 锁定相机；
2. 播放 8 秒初遇剪影；
3. 解锁 memory.rain.umbrella；
4. 存档 checkpoint.rain.complete；
5. 红伞覆盖画面，切换第三章。

### 5.9 第二章验收

- 静音和单声道均可完成；
- 地图褪色后仍保留玩家、已到站牌和红伞标记；
- 站牌踩错不清空正确前缀；
- 从任意可走区域 45 秒内能抵达一个有效线索；
- 过场期间暂停不会破坏 D1 状态。

### 6.5 谜题一：照片时序

- 谜题 ID：puzzle.life.photo_order
- 答案：1979 搬家 → 1992 桂花窗台 → 2001 银婚
- 操作：在 DOM 相册面板内用方向键选照片、确认拿起、方向键移动、确认放下

冗余线索：

| 照片 | 时间文字 | 非文字线索 |
| --- | --- | --- |
| 搬家 | 纸箱“1979” | 房间空、家具最新 |

### 6.6 谜题二：物件归位

- 谜题 ID：puzzle.life.object_placement
- 前置：照片时序完成
- 物件可按任意顺序放置
- 错误位置不消耗物品，物品自动返回手中

答案：

| 物品 | 位置 | 主要线索 | 纹理 |

| item.life.cassette | slot.life.radio | 收音机与相同哼唱 | 波纹 |

### 6.7 提示升级

提示时间从玩家第一次打开对应谜题面板开始。

| 时间 | 照片排序 | 物件归位 |

### 6.8 完成与出口

两组谜题完成后：

1. 空间从三个年代收束为当代客厅；
2. 林秀兰哼唱从录音带过渡为远处真实声音；
3. 解锁 journal.life.ordinary_days；
4. 存档 checkpoint.life.complete；
5. 客厅走廊延长，进入第四章。

### 6.9 第三章验收

- 不读取缺损名称仍可通过外形、位置与纹理解谜；
- 仅使用键盘可操作相册；
- 模拟色觉差异不影响答案判断；
- 三个物件任意顺序均不会破坏过场；
- 刷新后已归位物品与已播记忆保持一致。

## 7. 第四章：回家的路

### 7.1 地图参数

- 地图 ID：map.return_corridor
- 尺寸：36×20 格
- 退化阶段：D3
- 出生点：spawn.return.training = (4, 10)，朝右
- 结构：训练区 + 三个独立路口房间 + 出口

### 7.2 空间结构

~~~text
[训练区] → [路口一：看见] → [路口二：记住] → [路口三：相信] → [家门]
                ↺错路房间          ↺错路房间          ↺错路房间
~~~

每个错路房间复用当前章节已经见过的物件组合，但停留不超过三秒，随后返回当前路口。

### 7.3 安全教学

- 触发器：trigger.return.mapping_demo
- 输入锁定：是
- 演示时长：最短 3 秒，等待玩家确认后结束

标准模式：

1. 指南针与地砖箭头顺时针旋转；
2. 提示玩家按 W 或上键；
3. 角色安全向右移动一格；
4. 显示“方向变了。地上的箭头还在。”；
5. 玩家确认，写入 flag.return.mapping_learned。

低扰动模式：

1. 只让旧按键图标发生错位；
2. 角色仍按标准方向移动；
3. 显示“提示变得不可靠。地上的箭头还在。”。

完成后存档 checkpoint.return.training。

### 7.4 路线答案

路线以世界方向定义，输入系统负责变换。

| 路口 | 世界方向答案 | 标准模式物理输入 | 主要线索 | 通过检查点 |
| --- | --- | --- | --- | --- |
| 一 | 右 → 上 | 上 → 左 | 两枚完整地面箭头 | checkpoint.return.junction_1 |
| 二 | 下 → 右 | 右 → 上 | 第一枚箭头、红伞柄朝向 | checkpoint.return.junction_2 |
| 三 | 上 → 左 → 上 | 左 → 下 → 左 | 哼唱声源、窗帘与伞影 | checkpoint.return.junction_3 |

低扰动模式直接使用“世界方向答案”作为物理输入。

### 7.5 路口判定

- 每走入正确出口，RouteSystem 推进一步并关闭已完成入口；
- 走入错误出口时保持当前正确前缀；
- 完整路径正确才进入下一路口；
- 玩家在同一路口停留 45 秒未移动时，播放一次环境提示；
- 暂停、打开日记和浏览器失焦不计时。

### 7.6 错路与提示

| 错路次数 | 表现 |
| --- | --- |
| 1 | 当前红伞标记亮度提升 |
| 2 | 正确世界方向出现淡脚印 |
| 3 | 显示明确文字提示，不显示物理按键答案 |

三级提示示例：“先朝伞柄指向的方向走。方向键和脚步的关系已经转过一圈。”

### 7.7 章节完成

通过路口三后：

1. 映射仍保持 D3；
2. 玩家只能沿直线接近家门；
3. 自动解锁 journal.return.last_page；
4. 与门交互写入 checkpoint.return.complete；
5. 淡出并切换尾声。

### 7.8 第四章验收

- 映射全章只改变一次；
- 安全教学不允许失败；
- 暂停菜单始终使用标准方向；
- 标准和低扰动模式的世界路线完全一致；
- 静音状态下第三路口仍可凭窗帘与伞影通过；
- 错路不会清空正确前缀或返回章节开头。

## 8. 尾声

### 8.1 地图参数

- 地图 ID：map.home_ending
- 尺寸：复用 map.home 几何，移除谜题实体
- 退化阶段：D4
- 出生点：spawn.ending.hall = (25, 13)，朝左
- 林秀兰出生点：spawn.ending.xiulan = (30, 14)，朝左

### 8.2 流程

1. 恢复标准移动映射；
2. 按日记、背包、地图、目标的顺序淡出 HUD；
3. 林秀兰从玄关进入，在 (20, 12) 停下；
4. 玩家进入 trigger.ending.recognition 后锁定移动；
5. 播放结尾对白；
6. 激活长按牵手；
7. 成功后播放 6 秒固定演出；
8. 显示主题句；
9. 打开 DOM 科普入口。

### 8.3 长按区域

玩家无需精确站位。进入以林秀兰为中心、半径 160 像素的区域即可锁定到正确构图。长按取消不会重放对白，只恢复等待状态。

### 8.4 尾声验收

- HUD 消失不影响暂停和字幕；
- 长按可由无障碍设置替换为短按；
- 失焦会取消长按而不是直接完成；
- 玩家可跳过淡出动画，但不可跳过医疗免责声明；
- 科普页无需 Phaser Canvas 继续运行。
