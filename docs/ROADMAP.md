# 《记忆的缝隙》制作路线图

> 版本：v0.1
> 目标：8 周完成 20–30 分钟垂直切片
> 团队假设：3–5 人
> 技术基线：Phaser 3 + TypeScript + Vite

## 1. 交付目标

八周结束时提供一个可通过静态链接游玩的简体中文版本，完整包含：

- 开场设置与内容提示；
- 四章游戏流程与 D0–D4 退化；
- 标准模式和低扰动模式；
- 自动存档与章节恢复；
- 尾声与科普页；
- 键盘、静音、色觉冗余和减少动态效果路径；
- 医学与敏感性审核记录；
- 目标浏览器构建和测试证据。

## 2. 优先级

| 优先级 | 含义 | 处理规则 |
| --- | --- | --- |
| P0 | 缺失会使核心体验、退出或安全路径不可用 | 当前里程碑不得带入下一阶段 |
| P1 | 影响可理解性、情绪或无障碍完成率 | 发布候选前必须关闭 |
| P2 | 明显改善表现但不阻塞通关 | 时间允许时处理 |
| P3 | 原型后续方向 | 不进入八周承诺 |

## 6. Definition of Ready

任务进入开发前必须具有：

- 明确的稳定 ID；
- 可观察的完成结果；
- 前置依赖已经完成或已有替代方案；
- 所需文本、占位资产或地图坐标已存在；
- 无障碍替代路径已写明；
- 涉及医学内容时已标记审核责任。

## 7. Definition of Done

开发任务完成意味着：

- 功能满足文档验收，不只是“代码已写”；
- 对应单元或集成测试通过；
- 键盘和低扰动路径已人工检查；
- 不引入新的控制台错误、存档错误或焦点问题；
- 内容 ID 和资产 key 已登记；
- 相关文档随行为变化同步更新；
- 可由另一名成员从构建结果复现。

## 8. 变更控制

- 影响核心命题、章节顺序或 D0–D4 曲线：先更新 GDD；
- 影响状态边界、输入、存档、地图格式：先更新 TECHNICAL_DESIGN.md；
- 影响坐标、答案、检查点：更新 LEVEL_DESIGN.md；
- 影响对白或科普文字：更新 NARRATIVE_SCRIPT.md，并检查 MEDICAL_REVIEW.md；
- 影响尺寸、色彩、精灵或命名：更新 ART_BIBLE.md。

八周内新增功能必须说明替换掉哪个现有任务；不接受只增加范围、不调整时间或资源的变更。
