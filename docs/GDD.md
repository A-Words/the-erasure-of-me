# 《记忆的缝隙》（The Erasure of Me）游戏设计文档

> 文档版本：v0.1  
> 文档状态：垂直切片制作基线  
> 目标平台：PC 网页端  
> 目标时长：20–30 分钟  
> 核心技术：Phaser 3 + TypeScript + Vite  
> 语言：简体中文  
> 最后更新：2026-06-22

---

## 1. 项目概述

### 1.1 一句话介绍

玩家扮演患有轻度阿尔茨海默病的老人许志远，在逐渐失去地图、文字提示乃至熟悉操作方式的过程中，循着老伴林秀兰留下的日记、旧照片和一把红伞，拼凑两人共同生活的痕迹。

### 1.2 产品定位

| 项目 | 定义 |
| --- | --- |
| 类型 | 2D 俯视角叙事探索、轻度解谜 |
| 核心创新 | 游戏机制随剧情逐步退化，玩家需要重新理解原本熟悉的操作与信息 |
| 主题 | 阿尔茨海默病关注、家庭陪伴、认知障碍友好社会 |
| 目标玩家 | 16 岁以上叙事游戏玩家；关注父母与祖辈健康的青年及家庭成员 |
| 使用场景 | 浏览器独立试玩、公益活动展示、课程或比赛演示 |
| 单局时长 | 首次体验 20–30 分钟 |
| 商业模式 | 原型阶段免费、无广告、无内购 |
| 内容等级 | 无暴力、无惊吓；包含疾病、记忆衰退与情绪困扰相关内容提示 |

### 1.3 设计目标

1. 让玩家从“我知道该怎么做”逐步进入“我必须重新确认一切”的状态。
2. 让退化带来短暂不安，但始终保持公平、可推理和可完成。
3. 不把阿尔茨海默病表现为怪物、惩罚或人格消失，也不暗示疾病能够通过意志或解谜被治愈。
4. 在结尾把注意力从“找回全部记忆”转向“理解、尊重和陪伴仍然有意义”。
5. 给玩家一份简洁、可执行、不过度承诺的“早期表现与就医陪伴指南”。

### 1.4 非目标

- 不制作战斗、追逐、死亡、排行榜、开放世界或多结局。
- 不模拟所有认知障碍患者的共同经历；许志远仅代表一个虚构个体。
- 不将关卡中的认知练习包装为筛查、诊断、治疗或病程评估工具。
- 不要求玩家依靠反复试错承受不可预测的随机操作变化。
- 不在垂直切片阶段扩展完整发行、付费、多人或账号系统。

## 2. 主题与表达原则

### 2.1 核心命题

**记忆可能失去清晰的名称和顺序，但一个人曾经爱过、被爱过的痕迹不会因此失去价值。**

本作不以“恢复正常”作为胜利。玩家真正完成的是一段关系的重新确认：许志远最终无法说出林秀兰的姓名，却仍对她的声音、手势和掌心温度作出回应。

### 2.2 三项体验支柱

#### 机制即心境

退化不只出现在对白里，而直接作用于玩家已经学会的地图、背包文字、提示和移动方式。

#### 失序但不失信

规则可以改变，但不能欺骗玩家。每次改变必须同时满足：

1. 发生前有清晰的视听预兆；
2. 发生后至少保留两类不同感官的线索；
3. 同一关内规则保持固定；
4. 设置、暂停、退出和无障碍功能永不退化；
5. 玩家无法永久卡关。

#### 克制而温暖

视觉以褪色、留白、错位和声音远去表现认知变化，不使用怪物、血腥、跳吓或将患者“他者化”的语言。情绪终点不是悲剧反转，而是安静的陪伴。

### 2.3 用语规范

- 优先使用“患有阿尔茨海默病的人”“认知障碍患者及其家人”等以人为先的表达。
- 剧情人物不使用“累赘”“疯了”“没用了”等污名化判断；若测试中保留社会偏见台词，必须由作品明确反驳。
- 不使用“治愈记忆”“战胜阿尔茨海默病”“通关证明认知健康”等表述。
- “痴呆”仅在引用正式政策、医学名称或科普解释时使用，角色日常对白采用更自然且尊重的表达。

## 3. 世界观、人物与叙事

### 3.1 故事背景

故事发生在当代一座南方小城。现实中的许家是一个普通、有人长期生活过的两居室；记忆空间则从家中物件延伸为雨中的车站、旧屋和无法辨认方向的回家长廊。场景不是超自然世界，而是许志远对地点、时间和情绪的主观重组。

现实时间只经过一个清晨。游戏中的二十余分钟横跨许志远与林秀兰相识、成家和相伴至今的数十年。

### 3.2 主要人物

#### 许志远

- 年龄：72 岁。
- 身份：退休钟表维修师，已确诊轻度阿尔茨海默病。
- 性格：认真、要强、习惯用物件位置维持生活秩序；不愿承认自己需要帮助。
- 记忆锚点：钟声、修表工具的触感、林秀兰哼唱的旋律、红伞。
- 人物弧线：从执着于“必须想起正确答案”，转向允许自己接受提示和陪伴。

#### 林秀兰

- 年龄：70 岁。
- 身份：退休语文教师，许志远的妻子。
- 性格：耐心但并非永不疲惫；尊重许志远的选择，不把照护变成命令或考试。
- 叙事功能：通过日记和环境安排提供支持，但不替玩家完成谜题。
- 表达边界：她不是牺牲一切的完美照护者。科普页会同时肯定照护者休息、求助和获得支持的权利。

### 3.3 红伞意象

红伞是全作唯一持续保持高辨识度的暖色物体。

- 初遇时，林秀兰把伞向淋雨的许志远倾斜。
- 成家后，红伞出现在门口、合影和搬家纸箱旁。
- 记忆空间崩解时，红伞图形仍作为安全路标存在。
- 结尾现实中的旧伞已经褪成暗红，但许志远仍会伸手触摸伞柄。

红伞不是“恢复记忆的魔法物件”，而是长期生活形成的情感与动作线索。

### 3.4 叙事视角

- 全程贴近许志远的主观体验，不设置全知旁白解释病情。
- 日记由林秀兰以简短生活语言书写，不直接发布任务命令。
- 关键剧情使用场景行为、物件和短对白表达，避免长篇过场。
- 玩家可跳过文字动画，但不可跳过设置提示和医疗免责声明。

## 4. 核心玩法

### 4.1 玩家幻想

“我正在努力抓住一段重要的生活。熟悉的工具逐渐不再可靠，但我仍能借助声音、物件和另一个人的耐心找到回去的方向。”

### 4.2 玩家动词

| 动词 | 行为 | 默认输入 |
| --- | --- | --- |
| 移动 | 四方向行走 | WASD / 方向键 |
| 观察 | 高亮附近可关注物件 | Shift |
| 交互/确认 | 查看、拾取、放置、推进对白 | E / Enter / Space |
| 取消 | 关闭当前面板、返回上级 | Q / Backspace |
| 打开背包 | 查看持有物和记忆物件 | I |
| 打开日记 | 查看当前日记页 | J |
| 打开地图 | 查看当前空间 | M |
| 暂停 | 打开系统与无障碍菜单 | Esc |

鼠标可点击菜单和物品，但全流程不得要求鼠标输入。

### 4.3 核心循环

1. **进入空间**：观察房间、环境声音和当前可用 UI。
2. **寻找锚点**：发现红伞、钟声、日记语句、照片或熟悉物件。
3. **理解变化**：确认本阶段丢失了什么信息，以及哪些信息仍然可靠。
4. **重建联系**：通过路线、排序、辨识或放置谜题连接记忆碎片。
5. **获得片段**：解锁一段短场景或对白，并把记忆碎片收入日记。
6. **进一步退化**：进入下一阶段，原有辅助减少，但旧锚点继续发挥作用。

### 4.4 成功与失败

本作没有死亡和 Game Over。

- 谜题输入错误时，只恢复到该谜题最近的安全状态。
- 路线走错时，空间以自然回环方式把玩家送回最近路标，不播放失败音效。
- 连续两次无进展，环境线索增强；连续三次无进展，显示可关闭的明确提示。
- 玩家可从暂停菜单重置当前谜题或返回本章起点，不会丢失已完成章节。
- 提示不使用“错误”“失败”“你忘了”等责备语言，统一采用“再看看照片背面”“也许钟声指向另一边”等中性表达。

## 5. 机制退化系统

### 5.1 退化阶段

| 阶段 | 影响 | 保留信息 | 预兆 | 生效范围 |
| --- | --- | --- | --- | --- |
| D0 完整 | 无退化 | 全部 UI 与标准输入 | 无 | 第一章 |
| D1 地图褪色 | 未停留区域和道路逐步洇开，部分房间名消失 | 玩家位置、已到达地标、红伞标记、钟声方向 | 雷声、地图边缘出现水渍 | 第二章 |
| D2 文字缺失 | 背包名称从完整词语变为缺字和近似符号 | 物品图标、形状纹理、拾取地点、检查动画 | 字形轻微漂移，许志远停顿 | 第三章 |
| D3 方向错位 | 物理方向输入整体顺时针旋转 90° | 地面箭头、红伞路标、林秀兰哼唱方位、操作演示 | 指南针旋转、地砖箭头依次转向、屏幕短暂静止 | 第四章 |
| D4 安静留白 | 游戏性 HUD 隐去 | 场景、角色动作、确认键长按反馈 | 环境声收束，界面像纸屑般淡出 | 尾声 |

### 5.2 D3 固定映射

标准模式在第四章触发一次固定映射，此后直到尾声不再变化：

| 玩家按下 | 角色移动 |
| --- | --- |
| 上 / W | 右 |
| 右 / D | 下 |
| 下 / S | 左 |
| 左 / A | 上 |

映射改变前，游戏进入不可移动的三秒演示：

1. 指南针顺时针旋转；
2. 地面四枚方向图标依次亮起；
3. 游戏要求玩家按一次上键，角色以安全的一格距离向右移动；
4. 画面显示“方向变了。地上的箭头还在。”；
5. 玩家确认后正式开始本章。

任何时候打开暂停菜单，菜单导航都使用标准方向，不受 D3 影响。

### 5.3 低扰动模式

低扰动模式可在首次启动和暂停菜单中随时启用，启用后立即生效：

- D1 与 D2 保留，但降低模糊、漂移和文字闪动幅度；
- D3 不改变实际移动映射，只让画面中的旧按键提示发生错位；
- 地面箭头和林秀兰的哼唱仍用于路线判断；
- 所有非必要转场时长缩短，关闭屏幕震动、噪点和快速闪烁；
- 切换模式不会重置关卡或影响成就，因为原型不设置难度成就。

### 5.4 不可退化的安全层

以下功能永久稳定：

- Esc 暂停、退出和返回；
- 音量、字幕、字体、对比度、低扰动与减少动态效果设置；
- 焦点框、键盘菜单导航和确认/取消逻辑；
- 内容提示、医疗免责声明和外部求助链接；
- 存档反馈和浏览器关闭前的安全提示。

## 6. 关卡与流程

### 6.1 总体节奏

| 时间 | 章节 | 主要目标 | 新机制 | 情绪 |
| --- | --- | --- | --- | --- |
| 0:00–1:30 | 开场与设置 | 完成内容提示、无障碍选择 | 无 | 安静、日常 |
| 1:30–5:00 | 第一章：清晨的家 | 找到钥匙和日记 | 完整 UI 教学 | 熟悉、轻微迟疑 |
| 5:00–11:00 | 第二章：雨中的初遇 | 穿过车站找到红伞 | 地图褪色、声音导航 | 迷茫、好奇 |
| 11:00–18:00 | 第三章：共同生活 | 整理照片并归还生活物件 | 道具名称缺失 | 温暖、吃力 |
| 18:00–25:00 | 第四章：回家的路 | 适应方向错位并走出长廊 | 固定输入旋转 | 恐慌、重新建立信任 |
| 25:00–28:00 | 尾声 | 回应林秀兰、牵手 | HUD 消失、长按交互 | 释然、陪伴 |
| 28:00–30:00 | 科普与致谢 | 阅读或收藏指南 | 稳定 DOM 页面 | 理解、行动 |

玩家阅读速度较快时可在约 20 分钟完成；完整检查物件时不应超过 30 分钟。

### 6.2 开场与设置

#### 画面

黑屏中先听见钟表走动和厨房水声，随后出现标题。红伞靠在标题右侧，伞面有一小块暖色反光。

#### 必经信息

1. 内容提示：本作涉及认知衰退、迷路和家庭照护。
2. 体验声明：作品为虚构叙事，不代表所有患者经历。
3. 模式选择：标准模式 / 低扰动模式。
4. 字幕、字体大小、减少动态效果和音量快捷入口。

默认焦点为“开始”，但首次启动必须让玩家看到无障碍入口；无需强迫逐项设置。

### 6.3 第一章：清晨的家

#### 场景

现实中的客厅、卧室、厨房和玄关。空间尺度小，房门彼此可见。暖灰晨光与钟表声建立安全感。

#### 目标

找到出门钥匙，并带上林秀兰放在桌边的日记。

#### 流程

1. 玩家在卧室醒来，学习移动。
2. 检查床头合影，学习观察和交互；照片中可见红伞。
3. 打开地图，确认厨房、客厅、玄关的位置。
4. 在客厅拾取眼镜盒，学习背包；该物品不是解谜必需品。
5. 日记写着：“钥匙还在门口的小碗里。别急，我去买面，很快回来。”
6. 玩家在玄关蓝色圆碗中取得钥匙。
7. 拾取日记并尝试开门，雷声响起，场景过渡至记忆车站。

#### 教学约束

- 每个按键只强制教学一次。
- 玩家提前完成操作时立即跳过对应提示。
- 90 秒未找到钥匙时，蓝色小碗出现柔和轮廓；150 秒后日记自动翻到相关句子。

#### 完成条件

玩家持有钥匙与日记，并与玄关门交互。

### 6.4 第二章：雨中的初遇

#### 场景

1976 年夏天的旧车站。现实客厅的墙线延伸成站台，家具轮廓与候车长椅重叠。雨幕持续，但不遮挡可交互物。

#### 目标

依据钟声、旧车票和红伞标记，从站台走到钟表铺门口。

#### 退化

玩家第一次打开地图时仍能看到完整路线。雷声过后，未走过的道路名称被水渍晕开，只保留：

- 玩家当前位置；
- 已经抵达的站牌；
- 红伞图案；
- 钟楼方向的声波标记。

#### 谜题 A：站牌路线

旧车票背面画有“2 → 4 → 5”三个站号。玩家需要按顺序踩亮对应站牌旁的数字石板。

- 数字同时配有圆点数量，避免只依赖文字识别。
- 踩错时已完成序列不清零，错误石板只短暂变暗。
- 两次错误后，车票上的下一个数字和圆点轻微发亮。

#### 谜题 B：声音与地标

通过三次钟声判断钟表铺方向，同时沿路寻找两个红伞招牌。静音模式下，屏幕边缘出现与钟声同步的同心圆，地面积水也会向声源方向振动。

#### 记忆片段

钟表铺门口，年轻的林秀兰把红伞偏向许志远。画面不显示完整对白，只保留：

> “伞往你那边一点，别淋着。”

玩家取得记忆碎片“雨伞下的半边晴天”。

#### 完成条件

依次完成站牌路线并触碰钟表铺门口的红伞。

### 6.5 第三章：共同生活

#### 场景

三个时期的家叠在同一房间：搬家时的纸箱、种下桂花的窗台、结婚纪念日的餐桌。玩家每完成一个谜题，空间就稳定一部分。

#### 目标

将三张照片按时间排序，并把三件生活物品放回照片对应的位置。

#### 退化

背包名称依次从完整文字变为缺字：

- “木梳”变为“木＿”；
- “搪瓷杯”变为“＿瓷＿”；
- “录音带”变为波形符号。

物品图标、外形、纹理、检查动画和拾取地点始终保持一致。玩家观察物品时，许志远会用触觉描述代替名称，例如“齿很密，握柄上有一道修过的裂缝”。

#### 谜题 A：照片时序

三张照片分别为：

1. 搬进第一间家，纸箱上写有“1979”；
2. 窗台桂花第一次开花，孩子身高刻度旁写有“1992”；
3. 银婚纪念日，蛋糕牌写有“2001”。

玩家把照片放入从左至右的相册槽位。年份不是唯一线索，服装、家具新旧和桂花生长高度同时提供时序依据。

#### 谜题 B：生活物件归位

| 物件 | 对应位置 | 主要线索 | 冗余线索 |
| --- | --- | --- | --- |
| 木梳 | 搬家照片中的镜台 | 镜台轮廓 | 梳柄裂缝与照片一致 |
| 搪瓷杯 | 桂花窗台 | 杯身桂花图案 | 窗台留下圆形水印 |
| 录音带 | 纪念日晚餐桌 | 照片中的收音机 | 交互时播放同一段哼唱 |

颜色只作为辅助。木梳、杯子和录音带分别配有条纹、圆点、波纹三种纹理。

#### 记忆片段

每件物品归位后出现不超过八秒的生活切片：一起拆纸箱、擦去窗台泥土、在停电的晚上听录音带。三个片段合并为“共同生活不是一张照片，而是许多重复的小事”。

#### 提示升级

1. 90 秒无进展：相册空位显出年代轮廓。
2. 150 秒无进展：日记自动翻到相应生活记录。
3. 210 秒无进展：正确位置出现物件纹理，不直接显示名称。

#### 完成条件

照片顺序正确且三件物品全部归位。

### 6.6 第四章：回家的路

#### 场景

由许家走廊、车站通道和钟表铺后门拼接成的循环长廊。空间由三个连续路口组成，每个路口只允许前往四个相邻房间，不存在自由碰撞迷宫。

#### 目标

适应一次固定的方向错位，依靠地面箭头、红伞图案与林秀兰的哼唱通过三个路口。

#### 退化与教学

进入本章后执行第 5.2 节的三秒安全演示。玩家必须完成一次无风险输入练习，才进入第一个路口。

#### 三个路口

1. **路口一：看见**  
   地面明确画出两步路径，教学玩家按键与移动方向的对应关系。
2. **路口二：记住**  
   地面只保留第一步箭头，第二步需根据墙上的红伞朝向推理。
3. **路口三：相信**  
   地面箭头褪去。标准模式依据耳机左右声道中的哼唱方向和红伞柄影子前进；静音时以窗帘摆动和屏幕边缘波纹替代声音。

#### 错路处理

- 错误出口通向一个熟悉房间，行走三秒后自然回到当前路口。
- 第一次错路：红伞标记变亮。
- 第二次错路：地面出现一对淡脚印。
- 第三次错路：显示“跟着伞柄指向，再试一次”的明确提示。
- 不记录错误次数为负面统计，不播放刺耳反馈。

#### 记忆片段

通过第三个路口后，日记最后一页只剩一句：

> “如果你一时叫不出我的名字，也没关系。我会再告诉你。”

#### 完成条件

通过三个路口并与尽头的现实家门交互。

### 6.7 尾声

#### 表现

门打开后回到清晨的客厅。地图、背包、日记和提示逐项淡出。移动输入恢复标准，但玩家只需走到餐桌旁，不再安排谜题。

林秀兰提着面从门外进来。许志远看着她，字幕出现：

> 许志远：“我……你是……”  
> 林秀兰：“我是秀兰。别急，我回来了。”

不出现对白选项。林秀兰把手停在两人之间，留出等待空间。玩家长按任一确认键 1.5 秒，许志远才主动握住她的手；按住过程中只显示掌心逐渐变暖的动画，不显示按键文字。

最后一句字幕：

> “有些名字会远去，爱曾经来过的地方还留着温度。”

随后出现项目主题说明与“早期表现与就医陪伴指南”入口。玩家可立即退出，也可阅读、复制来源链接或重新开始。

## 7. UI、交互与无障碍

### 7.1 UI 分层

- Phaser Canvas：场景、角色、可交互物、天气、灯光与退化视觉效果。
- DOM HUD：地图、背包、日记、字幕、交互提示和目标。
- DOM 系统层：主菜单、暂停、设置、内容提示、科普页和致谢。

游戏规则状态不由 DOM 或 Phaser 精灵持有，二者只读取状态并发送语义化输入动作。

### 7.2 HUD 布局

| 区域 | 内容 | 显示规则 |
| --- | --- | --- |
| 左上 | 当前短目标，最多一行 | 目标更新时显示 4 秒，之后收起为小圆点 |
| 右上 | 小地图 | D0 常驻；D1 开始褪色；可用 M 展开 |
| 左下 | 背包入口与当前物品图标 | 拾取后显示 3 秒，I 打开完整抽屉 |
| 下中 | 情境交互提示 | 仅靠近物件时出现，避免遮挡角色 |
| 右下 | 日记入口 | 新页解锁时短暂发光，不使用红色通知点 |

常驻 HUD 覆盖面积不超过视口的 20%，屏幕中央和角色脚下保持清晰。

### 7.3 视觉可读性

- 正文字体最小 18 CSS px，默认 20 px；大字体模式为 24 px。
- 字幕最多两行，每行不超过 22 个汉字，可调整背景不透明度。
- 文字与背景对比度目标不低于 4.5:1。
- 可交互物不能只靠发光表示，同时提供轮廓、轻微动作或观察音。
- 所有颜色谜题必须同时具备形状、纹理或文字冗余。
- 支持 16:9 与 16:10；最小设计视口 1024×576，推荐 1366×768 以上。

### 7.4 动效与声音辅助

- 支持减少动态效果：关闭镜头轻晃、文字漂移、噪点和非必要缩放。
- 不使用每秒三次以上的闪烁。
- 重要声音线索必须有同步视觉替代；重要视觉线索应有声音或触觉式动画替代。
- 环境声、音乐、对白、界面音分设音量滑块。

### 7.5 键盘与焦点

- 所有 DOM 面板具有明确焦点框和顺序。
- 打开面板时冻结角色移动；关闭后焦点回到打开该面板的入口。
- 标准方向键只负责当前焦点层，绝不同时控制角色与菜单。
- 浏览器失焦时自动暂停，不继续计时或播放关键对白。

## 8. 美术方向

### 8.1 总体风格

采用低饱和手绘材质与轻微纸张颗粒。现实空间边缘清楚、比例稳定；记忆空间逐渐出现透视错位、缺口和重叠，但可通行区域必须保持高可读性。

### 8.2 色彩脚本

| 章节 | 主色 | 暖色锚点 | 退化表现 |
| --- | --- | --- | --- |
| 清晨的家 | 米灰、木棕、晨光蓝 | 红伞、蓝色钥匙碗 | 几乎无 |
| 雨中的初遇 | 雨蓝、旧纸黄 | 鲜红伞面 | 地图水渍、边缘晕染 |
| 共同生活 | 灰绿、旧木色 | 桂花黄、暗红伞 | 文字缺口、物件轮廓留白 |
| 回家的路 | 冷灰、深蓝 | 红伞图形 | 空间重复、方向图标旋转 |
| 尾声 | 柔和晨光、低饱和暖白 | 褪色红伞、手部肤色 | UI 安静淡出 |

红色只用于红伞和极少数情感焦点，不作为错误或危险颜色。

### 8.3 角色表现

- 俯视 3/4 角角色，四方向行走，每方向 4 帧。
- 许志远动作略慢但不夸张佝偻；停顿表现为观察和呼吸，不使用滑稽踉跄。
- 林秀兰的照护动作先停下等待，再伸手邀请，不强行拖拽。
- 记忆中的年轻角色减少面部细节，以姿态、伞和距离表达关系。

### 8.4 镜头

- 房间式柔性跟随，镜头不快速追赶。
- 正常移动不使用屏幕震动。
- 退化触发时最多进行 2% 的缓慢偏移，减少动态效果模式下完全关闭。
- 关键物件通过构图和局部光线引导，不依赖强制镜头锁定。

## 9. 音频方向

### 9.1 音乐

全作使用同一段“红伞主题”，由四个可分离声部组成：

1. 钢琴主旋律；
2. 木吉他和弦；
3. 轻柔弦乐持续音；
4. 钟表与雨声构成的节奏层。

随着机制退化，高频旋律和节奏层逐步减少，但低频和弦保持，象征具体记忆模糊而情感余韵仍在。尾声不是恢复完整主题，而是由林秀兰轻声哼出不完整旋律。

### 9.2 环境与提示音

- 清晨：钟表、烧水、远处街声。
- 车站：雨声、钟楼、站牌金属声。
- 共同生活：纸箱摩擦、窗边风声、录音带底噪。
- 回家长廊：脚步回声、哼唱、门锁轻响。
- UI：纸页、木片和呼吸感音色，不使用电子警报。

声音导航使用宽缓声像，不要求精确耳机定位；单声道模式下由视觉波纹完整替代。

### 9.3 语音范围

原型仅录制不超过 8 句林秀兰关键对白和 4 组许志远呼吸/迟疑声。全部语音配字幕，文字内容不依赖语气才能理解。

## 10. 科普与公益内容

### 10.1 定位

科普页名称固定为“早期表现与就医陪伴指南”，不使用“自测”“诊断测试”或“通关评分”。游戏内的排序、连接和辨识只服务于叙事体验。

首屏固定声明：

> 本页用于一般健康科普，不能替代专业筛查、诊断或治疗。如果你发现自己或家人的认知、情绪或日常能力持续发生变化并影响生活，请记录具体情况，并向正规医疗机构的相关专业人员咨询。

### 10.2 早期变化提示

以“值得留意的持续变化”而非症状清单考试呈现：

- 比过去更频繁地忘记近期事件或反复询问同一件事；
- 经常放错物品，难以沿原路寻找；
- 在熟悉地点迷路、混淆时间或地点；
- 完成熟悉任务、解决问题或作出决定变得困难；
- 跟随对话、理解表达或寻找词语变得困难；
- 视觉空间判断发生明显变化；
- 情绪、行为、兴趣或社交状态出现持续改变。

这些表现可能有多种原因，不能凭单一表现自行判断疾病。

### 10.3 建议行动

1. 记录变化出现的时间、频率、具体场景和对日常生活的影响。
2. 与本人平静沟通，避免在多人面前测试、指责或争辩“你怎么又忘了”。
3. 预约正规医疗机构评估，并携带既往病史、用药信息和观察记录。
4. 使用稳定日程、清晰标记、充足照明和紧急联系人信息改善日常安全。
5. 鼓励在能力允许的范围内继续参与熟悉活动和社会交往。
6. 照护者主动寻求家人、朋友、社区和专业支持，安排休息，不把疲惫视为失败。

### 10.4 游戏与公益机构合作

- 发布前由至少一名神经内科、老年医学、精神心理或认知障碍相关专业人员审核医学文字。
- 邀请患者家属或照护者进行敏感性阅读，但不要求患者承担未经支持的测试工作。
- 若与公益机构合作，机构标识、求助渠道和筛查信息必须由机构书面确认并注明适用地区与更新日期。
- 外部资源失效时不阻断游戏，只在科普页提示“链接待更新”。

1. [世界卫生组织：Dementia（2025-03-31 更新）](https://www.who.int/news-room/fact-sheets/detail/dementia)
2. [国家卫生健康委等15个部门：《应对老年期痴呆国家行动计划（2024—2030年）》](https://www.gov.cn/zhengce/zhengceku/202501/content_6996231.htm)

## 11. 技术设计

### 11.1 技术栈

- Phaser 3：场景、精灵、相机、动画、音频与输入接入。
- TypeScript：领域状态、配置与数据契约。
- Vite：开发服务器、构建和静态资源处理。
- HTML/CSS：HUD、菜单、设置、字幕与科普页。
- localStorage：单机轻量存档和设置。

### 11.2 逻辑分层

| 层 | 职责 | 禁止事项 |
| --- | --- | --- |
| Domain | 章节、谜题、背包、记忆、退化、提示与存档状态 | 不引用 Phaser 对象或 DOM |
| Application | 接收 InputAction，调度领域系统，发布状态事件 | 不持有精灵作为真值 |
| Phaser View | 渲染场景、角色、相机、动画、特效 | 不直接决定谜题是否完成 |
| DOM UI | 显示 HUD、菜单、字幕、设置与科普 | 不绕过领域系统修改进度 |
| Content/Data | 章节、对白、物品、谜题、退化配置 | 不以文件路径充当业务 ID |

建议场景为 BootScene、MenuScene、GameScene 和 EndingScene。章节切换由领域状态驱动，不为每个房间创建独立全局状态。

- 每次映射变化记录阶段 ID，不记录玩家具体按键内容。

### 11.6 存档

- 进入每章、完成每个主谜题和进入尾声时自动存档。
- 只保存可序列化领域状态，不保存 Phaser 精灵、Tween、相机或 DOM 引用。
- 设置数据与游戏进度分开保存。
- 存档解析失败时保留设置，提示玩家从最近完整章节重新开始。
- v1 只保留一个自动存档和“重新开始”入口。

- 目标：1366×768 的普通集成显卡笔记本上稳定 60 FPS；最低不低于 30 FPS。
- 首次可玩资源压缩后不超过 15 MB，总资源不超过 40 MB。
- 非当前章节音频和大图按章加载；切换章节时显示不超过三秒的主题过场。
- 页面失焦暂停更新与音频，恢复时先显示暂停层。
- 不要求联网运行，部署后所有核心内容可作为静态资源加载。

### 12.3 文本

- 可游玩正文控制在 6000 个汉字以内。
- 单次不可跳过说明不超过 80 个汉字。
- 日记每页不超过 120 个汉字。
- 所有文本维护内容 ID，便于后续本地化和医学复核。

---

## 13. 数据与观察

原型不接入广告、账号或第三方画像。正式公开版默认关闭遥测；组织测试时，只有在玩家明确同意后记录匿名事件。

建议事件：

- chapter_started / chapter_completed；
- puzzle_hint_shown，含谜题 ID 和提示等级；
- route_looped，含路口 ID，不含具体按键；
- accessibility_changed，含设置项，不记录系统健康信息；
- guide_opened / source_link_clicked；
- session_completed / session_abandoned。

不收集姓名、疾病史、自由文本、精确位置、按键序列或任何可用于推断个人认知状态的数据。测试结果只能用于改进游戏，不生成“认知评分”。

### 14.1 团队假设

## 15. 风险与应对

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 输入错位让玩家感到被戏弄 | 高 | 固定映射、安全演示、冗余线索、低扰动模式 |
| 退化表现污名化患者 | 高 | 以人为先语言、专业审核、家属敏感性阅读 |
| 小游戏被误解为医学测试 | 高 | 不评分、不诊断、科普页首屏声明、测试问卷验证 |
| 结尾被理解为“爱能治病” | 高 | 主角没有恢复姓名记忆，文案明确陪伴不等于治愈 |
| 声音谜题排除听障或静音玩家 | 高 | 所有声音线索提供同步视觉波纹 |
| 颜色谜题排除色觉障碍玩家 | 中 | 每种颜色同时绑定形状和纹理 |
| 浏览器失焦导致错过剧情 | 中 | 自动暂停，恢复时由玩家确认继续 |
| 文本和美术量超出八周 | 中 | 对白限字、复用同一空间变体、限制语音数量 |
| 情绪过重导致玩家中途退出 | 中 | 开场内容提示、暂停页随时退出、无追逐和惊吓 |

## 16. 测试与验收

### 16.1 功能测试

- WASD 与方向键均可完成全流程。
- 鼠标断开后仍可开始、设置、游玩、阅读指南和退出。
- 暂停菜单在 D3 中保持标准输入。
- 标准模式与低扰动模式可在任意检查点互相切换。
- 静音、关闭音乐和单声道条件下所有谜题可完成。
- 大字体不会遮挡关键交互或超出屏幕。
- 刷新、关闭页面和浏览器崩溃后可从最近检查点继续。
- 存档版本错误时不会白屏或破坏设置。

### 16.2 可用性测试

招募至少 15 名首次游玩者，覆盖主要青年玩家，并包含部分 45 岁以上家庭成员。患者与照护者参与需采用单独、知情且有支持的流程，不作为常规压力测试对象。

通过标准：

- 至少 80% 的首次游玩者在 20–30 分钟内无需外部攻略通关。
- 任一主谜题的明确三级提示触发率不超过 30%。
- D3 教学后，至少 80% 玩家能在两次尝试内通过第一个路口。
- 没有玩家因设置层随退化变化而无法退出或调整无障碍选项。

### 16.3 主题理解测试

通关后以不超过五题的匿名问卷验证：

1. 游戏是否表现为主角被治愈？正确理解应为“没有”。
2. 认知小游戏能否用于判断自己是否患病？正确理解应为“不能”。
3. 玩家能否说出至少一种支持方式，如记录变化、就医评估、耐心沟通或照护者求助？
4. 玩家是否认为游戏把患者表现为危险或失去人格？
5. 结尾最重要的变化是什么？

至少 80% 玩家应理解“疾病未被治愈，但陪伴仍有价值”；若任何玩家认为小游戏具有诊断作用，必须修改相关入口和文案后重新测试。

### 16.4 无障碍测试

- 纯键盘通关；
- 静音通关；
- 模拟常见色觉差异后通关；
- 低扰动与减少动态效果模式通关；
- 200% 浏览器缩放下完成菜单操作；
- 屏幕阅读器可读取设置、内容提示和科普页；游戏场景本体至少提供当前目标与关键交互文字。

### 16.5 浏览器视觉回归

每次候选发布构建都在 1366×768、1920×1080 和 1024×576 三种视口保存固定截图：

1. D0 第一间可操作房间；
2. D1 地图褪色后的车站；
3. D2 背包文字缺失状态；
4. D3 方向教学与第三个路口；
5. D4 牵手前的尾声；
6. 暂停菜单、低扰动设置和科普页。

截图用于检查角色脚底基线、可交互物轮廓、HUD 遮挡、文字对比度、退化前后差异和中央游戏区域是否清晰。测试过程中还需在移动、对白和打开背包三种状态下改变窗口尺寸，确认场景不重置、焦点不丢失、角色不穿出可行走区域。视觉问题按严重程度记录复现步骤，并标注由领域逻辑、Phaser 渲染、DOM UI 或资产管线负责。

### 16.6 医学与敏感性审核

发布前必须形成书面审核记录：

- 医学事实、术语、求助边界和来源是否准确；
- 是否存在疾病必然快速恶化、所有患者表现相同等误导；
- 是否把照护者神化或把疲惫描述为不够爱；
- 是否通过恐怖化、幼儿化或强制纠错制造戏剧效果；
- 是否明确说明游戏不是筛查和诊断工具。

## 17. 原型完成定义

只有同时满足以下条件，垂直切片才可标记为完成：

1. 四章、尾声、科普页和致谢能够从头到尾游玩。
2. D0–D4 退化曲线全部由配置驱动，关卡内没有随机输入变化。
3. 标准模式与低扰动模式均通过完整回归测试。
4. 键盘、静音和颜色冗余路径均能独立通关。
5. 15 人可用性测试达到 80% 无外部攻略完成率。
6. 医学顾问或合作机构完成科普内容审核。
7. 患者家属或照护者敏感性阅读中的高优先级问题全部关闭。
8. 目标浏览器无阻塞缺陷，最低性能目标达成。
9. 游戏中不存在诊断、治疗或“爱能治愈疾病”的暗示。
10. 所有权威链接在发布构建当日复核有效并注明更新时间。

## 18. 版本边界与后续方向

v0.1 只定义一个可独立传播的完整垂直切片。以下内容可在原型验证成功后评估，但不属于当前制作承诺：

- 英文及其他语言本地化；
- 移动端触屏版本；
- 更多患者个体与家庭关系视角；
- 与公益机构联合制作的地区化服务地图；
- 独立下载版和离线展览模式；
- 经专业团队设计、与游戏明确隔离的正规科普互动内容。

任何扩展都不得以增加挫败为代价强化疾病“真实感”，也不得把玩家表现或遥测数据转化为健康判断。
